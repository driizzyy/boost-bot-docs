Setup Guide

Step-by-step guide to get Driizzyy Boost Bot V3 up and running

1

Install Dependencies

Install required Python packages using pip:

pip install discord.py discord-webhook fastapi uvicorn httpx requests aiohttp websockets colorama pymongo tls-client pystyle fade
2

Configure Bot Settings

Edit config/config.json with your settings:

  • Bot Token: Your Discord bot token
  • OAuth2 Settings: Client ID, secret, redirect URL
  • Payment API: SellAuth API key and shop ID
  • Captcha Solver: CapMonster API key
  • Webhook URL: For logging and notifications
3

Setup Token Files

Add your tokens to the appropriate files:

  • data/user_tokens.txt - User tokens for boosting
  • data/1m.txt - 1-month boost tokens
  • data/3m.txt - 3-month boost tokens
  • data/onliner.txt - Tokens for onliner system
  • data/proxies.txt - HTTP proxies (format: http://user:pass@ip:port)
4

Configure Discord Settings

Set up your Discord server:

  • Create ticket categories (Support & Purchase)
  • Set up channel IDs in config for welcome, vouch, stock display
  • Configure role IDs for verified/unverified, customer, staff, admin
  • Set owner IDs in the owners_ids array
5

Setup OAuth2 Application

Configure Discord OAuth2:

  • Go to Discord Developer Portal
  • Create new application or use existing
  • Add redirect URL: http://your-domain:port/callback
  • Enable required scopes: identify, guilds.join, bot
  • Copy Client ID and Client Secret to config
6

Run the Bot

Start the bot using Python:

python main.py

The bot will start and display the banner. Verify all systems are operational in the console output.

7

Initial Configuration

Use slash commands to finalize setup:

  • /configpanel - Review and adjust settings
  • /ticketsystem - Deploy ticket panel
  • /suggests - Deploy suggestion panel
  • /livestock - Deploy stock display
  • /verificationpanel - Review OAuth2 settings

SellAuth Integration

The bot integrates with SellAuth for payment processing:

{
  "sellauth": {
    "enabled": true,
    "api_key": "your-api-key",
    "shop_id": "your-shop-id",
    "custom_field_name": "discord_invite"
  }
}

Webhooks are automatically processed for new orders, keys are generated and delivered.

Captcha Solving

CapMonster integration for hCaptcha solving:

{
  "captcha_solver": {
    "solve_captcha": true,
    "use": "capmonster",
    "api_key": "your-api-key",
    "max_retries": 10
  }
}

Automatically solves captchas during boost process with fallback retry system.

Proxy Configuration

Add proxies to data/proxies.txt (one per line):

http://username:password@proxy-server.com:8080
http://user2:pass2@proxy2.com:3128

Proxies are randomly rotated for each boost attempt and onliner connection.

Token Format

All token files use plain text format (one token per line):

MTA1MjM0NTY3ODkwMTIzNDU2Nw.GaBcDe.FgHiJkLmNoPqRsTuVwXyZ
MTA5ODc2NTQzMjEwOTg3NjU0Mw.GxYzWv.UtSrQpOnMlKjIhGfEdCbA

Tokens are validated before use. Failed tokens are logged to data/output/failed_boosts.txt.

Frequently Asked Questions

What is the current development status?

Driizzyy Boost Bot V3 is currently 98% complete and in final testing phase. Most features are fully functional and stable. The remaining 2% includes minor UI improvements and additional quality-of-life features.

Do I need proxies to use this bot?

Proxies are highly recommended but not required. The bot has a "proxyless" mode in config.json. However, using proxies significantly reduces the risk of IP bans and rate limits from Discord.

How many tokens do I need?

Each server boost requires 2 tokens. For 14 boosts (level 2), you need 28 tokens. The bot automatically manages token allocation and tracks usage in real-time.

Is this bot against Discord's Terms of Service?

This tool is designed for educational purposes and personal server management. Users are responsible for compliance with Discord's Terms of Service. We recommend reading Discord's policies before use.

Can I run this on a VPS or hosting service?

Yes! The bot includes hosting-friendly modes and has been tested on various VPS providers. Use /test_hosting_mode command to verify your hosting environment compatibility.
